## **一、优先推荐扩展的盘口（性价比高）**

### **1. 棒球 MLB ⭐⭐⭐⭐⭐ 首推**

- **为什么优先：**
  - **Polymarket 上 MLB 市场非常活跃，赛季内每天 10-15 场，流动性好**
  - **比赛节奏慢，单场 3 小时，套利窗口多（每局都可能触发盘口变动）**
  - **MLB StatsAPI 官方免费且几乎实时（**`statsapi.mlb.com`**），延迟 1-3 秒，是所有体育里数据最好拿的**
- **结算延迟特点：MLB 比赛结束后 Polymarket 结算通常要 1-2 小时，窗口很大**
- **数据源：MLB StatsAPI（官方）+ ESPN + SportsDataIO，三源冗余很容易**

### **2. 冰球 NHL ⭐⭐⭐⭐**

- **Polymarket 季后赛期间市场很活跃**
- **NHL 官方 API（**`api-web.nhle.com`**）免费、实时**
- **加时和点球大战是套利甜点：很多人不知道是否进入加时，盘口反应慢**
- **数据源生态和 NBA 类似，已有篮球架构可复用**

### **3. 美式橄榄球 NFL ⭐⭐⭐⭐**

- **Polymarket 上流动性最好的体育之一，单场 market 深度能到几十万刀**
- **一周只有周日+周一+周四几个比赛日，资金可以集中打**
- **数据源：ESPN / NFL 官方 / SportsDataIO 都很好拿**
- **注意：NFL 结算争议少（规则清晰），延迟相对可预测**

### **4. 网球（大满贯 + ATP/WTA 1000） ⭐⭐⭐⭐**

- **结算延迟非常大的运动之一，因为：**
  - **比赛时长不固定（3 小时-5 小时）**
  - **退赛、医疗暂停常见**
  - **Polymarket 经常等官方 ATP/WTA 数据更新**
- **数据源选择：**
  - **Sofascore 内部 API 是网球最快的源之一**
  - **官方 ATP/WTA 网站可爬**
  - **Tennis Abstract / Jeff Sackmann 数据偏统计，不适合实时**
  - **API-Sports tennis 付费可用**
- **套利窗口大：比赛刚结束时，Polymarket 价格经常停在 0.93-0.97，比球类还宽**

## **二、值得做但有难度的盘口**

### **5. UFC / MMA ⭐⭐⭐⭐（延迟最大，但场次少）**

- **结算延迟极大：比赛结束 → 官方宣判 → Polymarket 结算，常常 30 分钟以上**
- **单场结果二元清晰，套利逻辑最干净**
- **数据源：**
  - **UFCStats.com 官方**
  - **Tapology / Sherdog 爬虫**
  - **Twitter @ufc 官号通知极快（KO/Submission 一发生几秒内就发）**
  - **直播 OCR 也很有效（KO 画面识别）**
- **痛点：场次少（一周 1-2 个赛事），但单次套利空间大（经常 3-5%）**

### **6. 高尔夫（PGA / 大满贯） ⭐⭐⭐**

- **结算窗口超大：比赛持续 4 天，每洞都在变化**
- **Polymarket 上有"谁夺冠"市场，最后几洞经常出现套利**
- **数据源：**
  - **PGA Tour 官方 API（免费但要爬）**
  - **DataGolf（付费，专业级）**
  - **ESPN Golf scoreboard**
- **难点：不是简单"比分"，是 leaderboard + 剩余洞数推算，状态机更复杂**

### **7. 拳击（重大赛事时）⭐⭐⭐**

- **类似 UFC，但场次更少**
- **BoxRec 是行业标准数据源**
- **一年也就几场顶级 PPV 值得做，但每次都是大机会**

## **三、非传统体育（值得探索）**

### **8. 电竞 ⭐⭐⭐⭐**

- **Polymarket 上电竞市场在增长：LoL Worlds、Dota TI、CS Major、Valorant**
- **数据源极其丰富且免费：**
  - **HLTV API（CS）**
  - **Liquipedia（全项目）**
  - **Riot 官方 API（LoL）**
  - **Pandascore（付费聚合，电竞专用）**
  - **Abios（同上）**
- **结算延迟大：Polymarket 对电竞结算尤其慢，因为 UMA 验证者对电竞不熟，争议期长**
- **套利空间常常 5%+**
- **建议优先做 CS、LoL、Dota 这三个**

### **9. 板球 Cricket（如果你愿意学规则）⭐⭐⭐**

- **印度市场对应的 Polymarket 板球市场流动性其实不差**
- **Cricbuzz / ESPNCricinfo 数据源好用**
- **比赛时长极长（T20 也要 3-4 小时，Test 5 天），延迟窗口巨大**
- **门槛：规则复杂（DLS 制度、雨停补偿），状态机难写**

### **10. 赛车（F1 / NASCAR / MotoGP） ⭐⭐**

- **F1 在 Polymarket 上有市场但流动性一般**
- **F1 官方 timing data 可以拿（有第三方接口）**
- **难点：不是简单胜负，涉及围栏、安全车、DSQ 改判**
- **改判风险高，结算结果不稳定，慎入**

## **四、不推荐扩展的盘口**

- **赛马：结果改判频繁，Polymarket 市场少**
- **田径/游泳/奥运单项：场次稀疏，不值得搭基础设施**
- **板球之外的小众联赛：流动性太差**

## **五、扩展优先级建议**

**如果让我给你一个明确的扩展路线：**

**Plaintext**

```
当前：足球 + 篮球
  ↓
P1（立刻）：MLB + NFL  ← 北美四大里最容易接，复用现有架构
  ↓
P2（1-2周）：NHL + 网球 ← NHL 复用篮球架构，网球独立做
  ↓
P3（1个月）：UFC + 电竞（CS/LoL） ← 单次收益大但需要专门做
  ↓
P4（按需）：高尔夫 / 板球 ← 状态机复杂，看 Polymarket 流动性
```

## **六、给你的一个反向思路**

**除了"按运动扩展"，你还可以按盘口类型扩展，比如同一场比赛里：**

- **胜负盘（你现在在做的）**
- **大小分盘（Over/Under）：延迟更大，因为要算总分**
- **让分盘（Spread）：延迟也大**
- **球员个人数据盘（NBA 球员得分、MLB 投手三振等）：这块结算最慢，Polymarket 上越来越多**
- **赛季级市场（冠军、MVP、季后赛晋级）：不算 latency arb，但是另一个套利逻辑**

**球员个人数据市场是我最想提醒你的：**

- **比如"LeBron 今晚得分 > 25.5"**
- **比赛结束后，Polymarket 需要等官方 box score 确认，延迟比胜负盘还大**
- **数据源用 NBA Stats API、Baseball Savant 这种官方统计接口**
- **竞争者比胜负盘少很多，是蓝海**
